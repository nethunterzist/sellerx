# Sprint 7: Doc–Kod Eşlemesi ve Nihai Doc Mimarisi

Sprint 7'nin odak alanı: Hangi doc hangi kodu dokümante ediyor (doc→kod, kod→doc) ve docs yapısının nihai mimarisi. **Sadece doc yazılır; kod değiştirilmez.**

## Dosya Listesi

| Dosya | İçerik |
|-------|--------|
| [01-doc-to-code.md](01-doc-to-code.md) | Her sprint ve ana doc dosyası → dokümante ettiği kod alanı. |
| [02-code-to-doc.md](02-code-to-doc.md) | Kod alanı → bakılacak doc (ters indeks). |
| [03-doc-architecture.md](03-doc-architecture.md) | docs yapısı, navigasyon, güncelleme kuralları. |

## Kapsam

- **Kaynak:** Tüm sprint klasörleri ve docs/README.md.
- **Güncelleme:** Yeni sprint veya doc dosyası eklendiğinde 01, 02 ve 03 güncellenir; docs/README.md klasör yapısı güncellenir.

## Nasıl Kullanılır

- "Bu kod alanı nerede dokümante?" → 02-code-to-doc.md.
- "Bu doc neyi dokümante ediyor?" → 01-doc-to-code.md.
- "Doc yapısı ve güncelleme kuralı?" → 03-doc-architecture.md ve docs/README.md.
